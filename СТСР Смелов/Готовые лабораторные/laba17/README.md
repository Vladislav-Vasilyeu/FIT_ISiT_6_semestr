# Лабораторная работа 17. Redis

## Подготовка

1. Установить Redis локально или использовать облачный Redis.
2. Установить зависимости:

```bash
npm install
```

3. Для удаленного Redis создать файл `.env` рядом со скриптами и записать строку подключения:

```env
REDIS_URL=redis://default:password@host:port
```

Если облачный сервис требует защищенное подключение TLS, используйте `rediss://`:

```env
REDIS_URL=rediss://default:password@host:port
```

По умолчанию, если `.env` не создан, скрипты подключаются к `redis://localhost:6379`.

## Redis Cloud

Можно использовать официальный облачный сервис Redis Cloud:

1. Зарегистрироваться на сайте Redis Cloud.
2. Создать бесплатную базу данных.
3. В настройках базы взять `public endpoint`, порт, имя пользователя и пароль.
4. Составить строку подключения в формате `redis://username:password@host:port`.
5. Записать ее в `.env` как `REDIS_URL`.

Пример:

```env
REDIS_URL=redis://default:AbCdEf123@redis-12345.c1.eu-west-1-1.ec2.cloud.redislabs.com:12345
```

## Задание 06. Ответы на вопросы

**In-memory Database** - база данных, которая хранит основной рабочий набор данных в оперативной памяти. Благодаря этому операции чтения и записи выполняются очень быстро. Для защиты от потери данных такие системы обычно используют механизмы сохранения состояния на диск.

**Хэш-таблица** - структура данных, в которой значения хранятся по ключам. Ключ преобразуется хэш-функцией в индекс внутренней таблицы, поэтому поиск, добавление и удаление обычно выполняются за время, близкое к постоянному.

**Персистентность данных в Redis** обеспечивается сохранением данных на диск. Redis поддерживает RDB-снимки состояния, AOF-журнал команд изменения данных, а также комбинированный режим. RDB удобен для периодических резервных снимков, AOF лучше подходит для восстановления последних операций.

**set, get, del, getset**:

- `set key value` записывает строковое значение по ключу.
- `get key` возвращает значение ключа.
- `del key [key ...]` удаляет один или несколько ключей.
- `getset key value` атомарно возвращает старое значение ключа и записывает новое. В современных версиях Redis вместо него часто используют `set key value get`.

**incr, decr**:

- `incr key` увеличивает целочисленное значение ключа на 1.
- `decr key` уменьшает целочисленное значение ключа на 1.

**mset, mget**:

- `mset key value [key value ...]` записывает несколько пар ключ-значение одной командой.
- `mget key [key ...]` возвращает значения нескольких ключей одной командой.

**hset, hget**:

- `hset key field value` записывает поле внутри хэша.
- `hget key field` возвращает значение поля хэша.

**hmset, hmget**:

- `hmset key field value [field value ...]` записывает несколько полей хэша. Команда считается устаревшей, вместо нее используют `hset` с несколькими полями.
- `hmget key field [field ...]` возвращает значения нескольких полей хэша.

**exists** - проверяет наличие одного или нескольких ключей и возвращает количество существующих ключей.
