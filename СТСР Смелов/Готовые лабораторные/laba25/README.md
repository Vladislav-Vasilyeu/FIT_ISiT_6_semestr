# Лабораторная работа 25

## Задание 01. JSON-RPC сервер 25-01

Сервер реализован на Node.js без внешних зависимостей.

### Запуск

```bash
npm start
```

Адрес сервера:

```text
http://localhost:3000
```

В POSTMAN нужно отправлять `POST`-запросы на `/` с заголовком:

```text
Content-Type: application/json
```

### Реализованные процедуры

| Процедура | Описание |
| --- | --- |
| `sum(x1, x2, ..., xn)` | Возвращает сумму всех параметров |
| `mul(x1, x2, ..., xn)` | Возвращает произведение всех параметров |
| `div(x, y)` | Возвращает `x / y` |
| `proc(x, y)` | Возвращает `x / y * 100` |

### Примеры запросов

#### sum

```json
{
  "jsonrpc": "2.0",
  "method": "sum",
  "params": [10, 20, 30],
  "id": 1
}
```

Ответ:

```json
{
  "jsonrpc": "2.0",
  "result": 60,
  "id": 1
}
```

#### mul

```json
{
  "jsonrpc": "2.0",
  "method": "mul",
  "params": [2, 3, 4],
  "id": 2
}
```

Ответ:

```json
{
  "jsonrpc": "2.0",
  "result": 24,
  "id": 2
}
```

#### div

```json
{
  "jsonrpc": "2.0",
  "method": "div",
  "params": [10, 2],
  "id": 3
}
```

Ответ:

```json
{
  "jsonrpc": "2.0",
```

#### proc


Ответ:


## Задание 02. Ответы на вопросы

### Что такое JSON-RPC?

JSON-RPC - это простой протокол удаленного вызова процедур, в котором запросы и ответы передаются в формате JSON. Клиент отправляет имя метода, параметры и идентификатор запроса, а сервер выполняет метод и возвращает результат или ошибку.

### Опишите формат RPC-запроса.

JSON-RPC 2.0 запрос содержит поля:

```json
{
  "jsonrpc": "2.0",
  "method": "methodName",
  "params": [],
  "id": 1
}
```

`jsonrpc` указывает версию протокола, `method` содержит имя вызываемой процедуры, `params` содержит параметры, `id` нужен для связи ответа с запросом.

### Позиционные параметры запроса.

Позиционные параметры передаются массивом. Их смысл определяется порядком следования.

Пример:

```json
{
  "jsonrpc": "2.0",
  "method": "div",
  "params": [10, 2],
  "id": 1
}
```

### Именованные параметры запроса.

Именованные параметры передаются объектом. Их смысл определяется именами свойств.

Пример:

```json
{
  "jsonrpc": "2.0",
  "method": "div",
  "params": {
    "x": 10,
    "y": 2
  },
  "id": 1
}
```

### Опишите формат RPC-ответа.

Успешный JSON-RPC ответ содержит версию протокола, результат и идентификатор исходного запроса:

```json
{
  "jsonrpc": "2.0",
  "result": 5,
  "id": 1
}
```

Если произошла ошибка, вместо `result` возвращается объект `error`:

```json
{
  "jsonrpc": "2.0",
  "error": {
    "code": -32601,
    "message": "Method not found"
  },
  "id": 1
}
```

### Что такое RPC-уведомление?

RPC-уведомление - это запрос без поля `id`. Сервер выполняет процедуру, но не отправляет ответ клиенту.

Пример:

```json
{
  "jsonrpc": "2.0",
  "method": "sum",
  "params": [1, 2, 3]
}
```

### Что такое пакет RPC-запросов?

Пакет RPC-запросов - это массив из нескольких JSON-RPC запросов, отправленный одним HTTP-запросом. Сервер обрабатывает каждый элемент массива и возвращает массив ответов.
