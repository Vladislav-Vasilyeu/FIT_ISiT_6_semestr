# Лабораторная работа 22

## Цель

Создать CA-сертификат, сформировать запрос на сертификат Resource, выпустить сертификат Resource и разработать приложение, принимающее GET-запросы по HTTPS.

## Задание 01 CA

Создан CA-сертификат с общим именем:

```text
CN = CA-LAB22-XYZ
```

Файлы CA:

- закрытый ключ: `certs/ca-lab22-xyz.key`
- сертификат: `certs/ca-lab22-xyz.crt`

CA получает от Resource запрос `certs/rs-lab22-abc.csr` и выпускает сертификат Resource.

## Задание 02 Resource

Создан запрос на сертификат с общим именем:

```text
CN = RS-LAB22-ABC
```

В сертификате Resource указаны домены:

```text
DNS:LAB22-ABC
DNS:ABC
```

Дополнительно добавлены `localhost` и `127.0.0.1` для удобной локальной проверки в браузере.

Файлы Resource:

- закрытый ключ: `certs/rs-lab22-abc.key`
- запрос на сертификат: `certs/rs-lab22-abc.csr`
- сертификат: `certs/rs-lab22-abc.crt`

CA-сертификат импортируется в доверенные корневые центры сертификации Windows командой:

```powershell
.\scripts\import-ca.ps1
```

## Приложение 22-01

Разработано приложение `22-01`, принимающее GET-запросы по HTTPS-протоколу.

Запуск:

```powershell
node .\22-01\server.js
```

Также запуск доступен через npm:

```powershell
npm start
```

Адрес для проверки:

```text
https://localhost:8443/
```

Проверочный JSON-эндпоинт:

```text
https://localhost:8443/status
```

## Вывод

В ходе лабораторной работы был создан собственный центр сертификации, выпущен сертификат для HTTPS-ресурса и разработано приложение, работающее по HTTPS.
