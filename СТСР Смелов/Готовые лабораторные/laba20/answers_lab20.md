# Ответы на вопросы к лабораторной работе 20

## 1. Шаблонизация

Шаблонизация - это способ формирования HTML-страницы на сервере по заранее подготовленному шаблону и данным приложения. Сервер берет шаблон, подставляет в него значения из контекста и отправляет клиенту готовый HTML.

## 2. Шаблон (template)

Шаблон - это файл с HTML-разметкой и специальными выражениями шаблонизатора. В Handlebars такие выражения записываются в двойных фигурных скобках, например `{{name}}`. При рендеринге они заменяются реальными данными.

## 3. Контекст шаблона (template context)

Контекст шаблона - это объект данных, который передается в шаблон при рендеринге. Например, при вызове `res.render('home', { contacts: data.contacts })` контекстом является объект с полем `contacts`; шаблон может обращаться к нему через `{{contacts}}`, `{{#each contacts}}` и другие конструкции.

## 4. Макет (layout)

Макет - это общий каркас страницы, который используется несколькими представлениями. Обычно в макет помещают `<!doctype html>`, тег `head`, подключение CSS/JS, шапку и место для содержимого конкретной страницы. В `express-handlebars` содержимое страницы вставляется в макет через `{{{body}}}`.

## 5. Частичное представление (partial-view)

Partial-view - это маленький переиспользуемый фрагмент шаблона. Его удобно применять для повторяющихся элементов интерфейса, например для строки телефонного справочника. В Handlebars partial подключается так: `{{> row contact=this}}`.

## 6. Хелпер (helper)

Хелпер - это функция, которую можно вызывать из шаблона. Хелперы используют для форматирования данных, условий, генерации небольших фрагментов HTML или повторяющейся логики. В данной лабораторной helper используется для кнопки `Отказаться`.

## 7. Встроенные хелперы Handlebars

Основные встроенные хелперы Handlebars:

- `{{#if}}` - условный вывод, если значение истинно.
- `{{#unless}}` - условный вывод, если значение ложно.
- `{{#each}}` - перебор элементов массива или объекта.
- `{{#with}}` - смена текущего контекста внутри блока.
- `{{lookup}}` - получение значения по динамическому имени свойства.
- `{{log}}` - вывод отладочной информации.

## 8. Статический файл

Статический файл - это файл, который сервер отдает клиенту без генерации содержимого на лету. К таким файлам относятся CSS, клиентский JavaScript, изображения, шрифты. В Express для их раздачи используется middleware `express.static(...)`.

## 9. HTTP-заголовки Cache-Control, ETag, Last-Modified, If-Modified-Since, If-None-Match

`Cache-Control` задает правила кэширования ответа: можно ли кэшировать файл, как долго он считается свежим, нужно ли повторно проверять его у сервера.

`ETag` - уникальный идентификатор версии ресурса. Если файл изменился, обычно меняется и его ETag.

`Last-Modified` показывает дату и время последнего изменения ресурса.

`If-Modified-Since` отправляется клиентом при повторном запросе. Клиент сообщает серверу дату из прошлого `Last-Modified`; если ресурс не менялся, сервер может вернуть `304 Not Modified`.

`If-None-Match` отправляется клиентом вместе с ранее полученным `ETag`. Если текущий ETag совпадает, сервер может вернуть `304 Not Modified` без тела ответа.

## 10. Свойство `__dirname` объекта global

`__dirname` в Node.js содержит абсолютный путь к каталогу, где расположен текущий исполняемый файл модуля. Его часто используют для построения надежных путей к файлам проекта: `path.join(__dirname, 'public')`, `path.join(__dirname, 'views')`.

## 11. Метод `response.download()`

`response.download(path[, filename][, options][, callback])` отправляет файл клиенту как скачиваемый. Express устанавливает заголовок `Content-Disposition: attachment`, поэтому браузер обычно предлагает сохранить файл. Параметр `filename` позволяет задать имя файла, которое увидит пользователь.

## 12. Метод `response.attachment()`

`response.attachment([filename])` настраивает ответ как вложение для скачивания. Метод выставляет `Content-Disposition: attachment`, а если передано имя файла, Express также определяет подходящий `Content-Type`. В отличие от `download()`, сам файл этот метод не отправляет; он только настраивает заголовки ответа.

## 13. Заголовок `Content-Disposition: attachment`

`Content-Disposition: attachment` сообщает браузеру, что содержимое ответа нужно обработать как вложение, то есть обычно скачать, а не открыть прямо в окне. Дополнительно может указываться имя файла: `Content-Disposition: attachment; filename="report.txt"`.
