# Теоретические сведения

## Принципы разработки приложения по паттерну MVC

MVC (Model-View-Controller) - это архитектурный паттерн, который разделяет приложение на три логические части:

- Model - модель данных и бизнес-логика приложения. Она отвечает за хранение, получение, изменение и проверку данных. В данном проекте роль простой модели выполняет массив `users` в файле `controllers/controller.js`.
- View - представление, то есть слой, который отвечает за отображение результата пользователю. В веб-API вместо HTML-представления часто используется JSON-ответ. В этом приложении ответы формируются методами `res.json(...)`, `res.status(...).json(...)` и `res.send()`.
- Controller - контроллер, который принимает запрос, обращается к данным, выполняет нужное действие и формирует ответ. В проекте контроллер описан в `controllers/controller.js`: методы `getAll`, `getOne`, `create`, `update`, `delete` обрабатывают операции над пользователями.

Главная идея MVC - разделить ответственность между частями приложения. Маршрутизатор определяет, какой запрос пришел, контроллер выполняет действие, модель хранит и изменяет данные, а представление возвращает результат клиенту. Благодаря этому приложение легче расширять: можно добавить новый маршрут, новый метод контроллера или заменить способ хранения данных без полной переработки программы.

## Что описывает таблица маршрутизации

Таблица маршрутизации описывает соответствие между HTTP-запросом и действием приложения. В файле `routes/routeTable.js` каждая запись содержит:

- `method` - HTTP-метод запроса: `GET`, `POST`, `PUT`, `DELETE`;
- `path` - путь запроса, например `/users` или `/users/:id`;
- `controller` - имя контроллера, который должен обработать запрос;
- `action` - метод контроллера, который нужно вызвать.

Например, запись

```js
{ method: 'GET', path: '/users/:id', controller: 'controller', action: 'getOne' }
```

означает: если клиент отправил `GET`-запрос на адрес `/users/1`, приложение должно вызвать действие `getOne` из контроллера.

## Что описывает таблица контроллеров

Таблица контроллеров описывает, какие контроллеры существуют в приложении и где находятся их обработчики. Она связывает имя контроллера из таблицы маршрутизации с конкретным программным модулем.

В текущем проекте отдельной таблицы контроллеров нет, потому что используется только один контроллер:

```js
const controller = require('./controllers/controller');
```

Но в более крупном приложении таблица контроллеров могла бы выглядеть так:

```js
const controllerTable = {
    controller: require('./controllers/controller'),
    productController: require('./controllers/productController')
};
```

Тогда маршрутизатор мог бы брать значение `controller` из таблицы маршрутизации, находить соответствующий модуль в таблице контроллеров и вызывать нужное действие.

## Назначение маршрутизатора

Маршрутизатор принимает входящий HTTP-запрос и определяет, какой обработчик должен его выполнить. В файле `19-01.js` маршрутизатор создается на основе таблицы маршрутизации:

```js
routeTable.forEach(route => {
    const ctrl = controller;
    const action = ctrl[route.action];

    app[route.method.toLowerCase()](route.path, action);
});
```

Назначение маршрутизатора:

- прочитать записи из таблицы маршрутизации;
- зарегистрировать маршруты в Express;
- связать HTTP-метод и URL с методом контроллера;
- передать управление нужному обработчику при получении запроса.

Таким образом, маршрутизатор является связующим звеном между клиентским запросом и логикой контроллера.
