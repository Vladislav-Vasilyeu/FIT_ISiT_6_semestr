# Ответы на контрольные вопросы

## GraphQL

GraphQL - язык запросов к API и среда выполнения этих запросов. Клиент сам указывает, какие поля ему нужны, а сервер возвращает данные строго в запрошенной структуре.

## Схема GraphQL

Схема GraphQL - описание типов данных, запросов, мутаций и подписок, которые поддерживает сервер. Схема задает контракт между клиентом и сервером.

## SDL GraphQL

SDL расшифровывается как Schema Definition Language. Это язык описания GraphQL-схемы в текстовом виде: `type`, `input`, `enum`, `interface`, `union`, `schema`.

## Resolver GraphQL

Resolver - функция-обработчик поля схемы. Она получает аргументы запроса, обращается к источнику данных и возвращает значение для соответствующего поля.

## Query GraphQL

Query - операция чтения данных. В лабораторной к query относятся `getFaculties`, `getTeachers`, `getPulpits`, `getSubjects`, `getTeachersByFaculty`, `getSubjectsByFaculties`.

## Mutation GraphQL

Mutation - операция изменения данных. Она используется для добавления, обновления и удаления записей. В лабораторной это `setFaculty`, `setTeacher`, `setPulpit`, `setSubject`, `delFaculty`, `delTeacher`, `delPulpit`, `delSubject`.

## Subscription GraphQL

Subscription - операция подписки на события сервера. Она позволяет клиенту получать обновления в реальном времени, обычно через WebSocket.

## Context GraphQL

Context - общий объект, который передается resolver-функциям во время выполнения запроса. В него обычно помещают пользователя, права доступа, подключение к БД, логгер или другие общие зависимости.

## Схема работы модуля `graphql`

Модуль `graphql` получает текст запроса, проверяет его по схеме, строит план выполнения и вызывает resolver-функции для запрошенных полей. После выполнения он собирает результат в JSON и возвращает клиенту объект с `data` или `errors`.

## Компоненты схемы

`subscription` описывает операции получения событий от сервера в реальном времени.

`interface` задает общий набор полей, который обязаны реализовать несколько типов.

`enum` задает ограниченный набор допустимых значений.

`fragment` позволяет повторно использовать одинаковый набор полей в нескольких местах запроса.

`union` объединяет несколько типов в один возможный результат, когда поле может вернуть объекты разных типов.
