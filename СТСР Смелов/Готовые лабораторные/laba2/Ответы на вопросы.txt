 Вот ответы на ваши вопросы по Node.js:

## 19. Основные свойства сервера Node.js

- **Асинхронность и неблокирующий ввод/вывод** — операции выполняются без блокировки основного потока
- **Однопоточность с событийным циклом (Event Loop)** — обработка множества соединений через один поток
- **Высокая производительность** — благодаря движку V8 и асинхронной архитектуре
- **Масштабируемость** — легко масштабируется горизонтально (кластеризация)
- **Кроссплатформенность** — работает на Windows, macOS, Linux
- **Модульность** — система модулей CommonJS для организации кода
- **Обширная экосистема** — огромное количество пакетов через npm

## 20. Что такое npm?

**npm (Node Package Manager)** — это:
- Менеджер пакетов для Node.js
- Реестр (registry) JavaScript-пакетов — крупнейший в мире репозиторий open-source кода
- Инструмент командной строки для:
  - Установки зависимостей (`npm install`)
  - Управления версиями пакетов
  - Запуска скриптов (`npm run`)
  - Публикации собственных пакетов

## 21. Назначение HTTP-заголовка Content-Type

**Content-Type** указывает **тип и формат данных** в теле HTTP-сообщения (запроса или ответа).

**Формат:** `Content-Type: тип/подтип; параметры`

**Примеры:**
- `text/html; charset=utf-8` — HTML-документ
- `application/json` — данные в формате JSON
- `text/plain` — простой текст
- `image/png` — изображение PNG

**Назначение:** Позволяет клиенту и серверу корректно интерпретировать передаваемые данные.

## 22. Назначение функции require

**require** — функция для **подключения модулей** в Node.js (система CommonJS).

**Возможности:**
```javascript
const http = require('http');        // Встроенный модуль
const fs = require('fs');            // Встроенный модуль
const express = require('express');  // Модуль из node_modules
const myModule = require('./myModule'); // Локальный модуль
```

**Что делает:**
- Загружает модуль по указанному пути или имени
- Возвращает экспортированные данные (объект, функцию и т.д.)
- Кэширует загруженные модули для повторного использования

## 23. Модуль Node.js

**Модуль** — это **отдельный файл JavaScript**, содержащий связанный между собой код (функции, объекты, классы), который можно переиспользовать в других файлах.

**Особенности:**
- Каждый файл — отдельный модуль со своей областью видимости
- Экспорт через `module.exports` или `exports`
- Импорт через `require()`
- Помогает организовать код, разделяя его на логические блоки

## 24. Node.js built-in modules (Core modules)

**Встроенные (ядровые) модули** — модули, **поставляемые вместе с Node.js**, не требующие установки через npm.

**Примеры:**
- `http` / `https` — создание веб-серверов
- `fs` — работа с файловой системой
- `path` — работа с путями файлов
- `os` — информация об операционной системе
- `events` — работа с событиями
- `stream` — потоковая обработка данных
- `crypto` — криптографические функции

Подключаются напрямую: `const module = require('module_name')`

## 25. Модуль для работы с HTTP

**Модуль `http`** (и `https` для защищенного соединения)

```javascript
const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Hello World');
});

server.listen(3000);
```

## 26. Модуль для работы с файловой системой

**Модуль `fs` (File System)**

**Основные возможности:**
- Чтение файлов: `fs.readFile()`, `fs.readFileSync()`
- Запись файлов: `fs.writeFile()`, `fs.appendFile()`
- Работа с директориями: `fs.mkdir()`, `fs.readdir()`, `fs.rmdir()`
- Информация о файлах: `fs.stat()`, `fs.access()`
- Переименование/удаление: `fs.rename()`, `fs.unlink()`

**Пример:**
```javascript
const fs = require('fs');

// Асинхронное чтение
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});
```
